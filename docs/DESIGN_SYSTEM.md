# StayFinder AI — Design System

Source of truth is [`css/main.css`](../css/main.css). This doc explains the
tokens and components for anyone designing new screens (Figma or code) so
they stay consistent with what's shipped. Machine-readable tokens live in
[`design-tokens.json`](../design-tokens.json) — import that into Figma via
the **Tokens Studio** plugin to get matching Figma Variables.

## Look & feel

A warm, dim "coastal lodge at night" palette — espresso and bark browns,
gold/amber accents, a teal used sparingly for positive/live status. Display
type (Cormorant Garamond) for titles and prices gives it a boutique-travel
feel; DM Sans carries all UI text.

## Color

| Token | Hex | Use |
|---|---|---|
| `--espresso` | `#130e08` | App background |
| `--bark` | `#1e1610` | Panels, chat bubbles, header |
| `--driftwood` | `#2c2118` | Cards, inputs, raised surfaces |
| `--mahogany` | `#3d2e1e` | User avatar background |
| `--gold` | `#c8972a` | Primary accent — buttons, borders, active states |
| `--gold-light` | `#e8b94a` | Primary accent, emphasis text |
| `--amber` | `#d4793a` | Secondary accent, gradients with gold |
| `--cream` | `#f0e6d3` | Primary text |
| `--cream-dim` | `#a89880` | Secondary text |
| `--cream-faint` | `#5a4e40` | Tertiary text, placeholders, disabled |
| `--sand` | `#e8d5b8` | Text on user (gold-tinted) bubbles |
| `--teal` | `#2a8a7a` | Status accent |
| `--teal-light` | `#3ab89f` | "Live"/positive accent — status dot, best-value badges |

Borders are almost never solid — they're `rgba(200,151,42, 0.1–0.5)` (gold at
low opacity) so panels read as one dark surface with a soft edge, not boxed.

## Typography

- **Display** — `Cormorant Garamond` (Google Fonts): welcome title, listing
  titles, prices, modal headings. Weights 300–600, used mostly at 400–500.
- **Body** — `DM Sans`: everything else. Weight 300 is the default body
  weight (deliberately light or the UI feels heavy against the dark bg).

## Spacing & radius

- `--radius`: 14px — cards, bubbles, modals.
- `--radius-sm`: 8px — small chips, buttons, inputs inside cards.
- Chat bubbles use an asymmetric radius (`border-bottom-left-radius: 4px` for
  AI, `border-bottom-right-radius: 4px` for user) for a speech-bubble tail
  effect without an actual tail.

## Core components

- **Buttons** — `.btn-ghost` (bordered, transparent) for header actions;
  gradient `gold → amber` fill for primary actions (send, search, compare).
- **Chat bubble** — `.bubble-ai` / `.bubble-user`, max-width 78%, 14.5px text.
- **Listing card** — `.listing-card`: title + price header, location, rating,
  colored tags (`tag-gold` / `tag-green` / `tag-amber`), and a
  `.listing-links` row of real "Search Airbnb / Booking.com" deep links —
  StayFinder never claims to book, only to point you to a real search.
- **Reply chips** — `.reply-chip`: tappable short-answer suggestions rendered
  above the input, generated by the AI each turn via the `suggest_replies`
  tool.
- **Sidebar preferences** — label/value pairs (`.pref-label` uppercase 10px
  + `.pref-value`), a progress bar showing profile completeness, and
  amenity/activity tags.
- **Map panel** — slide-in on desktop (right, 420px), slide-up full-screen on
  mobile (≤900px). Falls back to a styled list (`showMapPlaceholder`) when no
  Mapbox token is configured, instead of a broken map.
- **Compare modal** — a table of up to 3 selected listings, side by side,
  with "Best price" / "Top rated" win badges.
- **Mic button** — `.mic-btn`, pulses teal via `.listening` while
  Web Speech API is capturing audio.

## Responsive breakpoints

- `≤1100px` — map panel narrows (420px → 360px).
- `≤900px` — layout collapses to a single column; sidebar becomes an
  off-canvas drawer (`.sidebar.open`); map becomes full-screen.
- `≤480px` — tighter padding/type scale for small phones.

## Accessibility notes

- The message log (`#messages`) is `role="log" aria-live="polite"` so screen
  readers announce new AI replies.
- All interactive icons (mic, send, close buttons) have `aria-label`.
- Every assistant message gets a 🔊 read-aloud button (`SpeechSynthesis`) for
  low-literacy or hands-busy use.
- Body copy from the AI is written at a 5th–6th grade reading level by the
  system prompt (see `api/chat.js`) — a content rule as much as a visual one.
